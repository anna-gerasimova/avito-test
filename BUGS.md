# Баг-репорты

## Баг 1

**Заголовок:** Не работает фильтрация по статусу «Done».

**Описание:** При выборе фильтрации по статусу «Done» список задач не фильтруется.

**Предусловие:** Есть задачи со статусом «Done».

**Окружение:** Windows 10, Chrome 143, разрешение 1920x1080.

**Шаги воспроизведения:**
1. В списке задач нажать на выпадающее меню «Статус».
2. Выбрать «Done».
3. Дождаться результата.

**Ожидаемый результат:** После выбора статуса «Done» в списке задач отображаются задачи с нужным статусом.

**Фактический результат:** После выбора статуса «Done» в списке задач не отображаются задачи с нужным статусом.

**Приоритет:** medium.

**Дополнительные материалы:** Имя файла «Статус»

## Баг 2

**Заголовок:** При редактировании поля «Название» в задаче и нажатия кнопки «Обновить» – текст не меняется.

**Описание:** При редактировании уже существующей задачи изменения в поле «Название» и нажатии кнопки «Обновить» – текст не меняется, оставаясь таким же.

**Предусловие:** Создана задача.

**Окружение:** Windows 10, Chrome 143, разрешение 1920x1080.

**Шаги воспроизведения:**
1. В списке задач нажать на любую задачу, например, «Реализация новой галереи изображений».
2. Дождаться открытия задачи
3. Внести любые изменения в названии задачи, например, «Реализация НОВОЙ галереи изображений».
4. Нажать на кнопку «Обновить»
5. Дождаться результата.

**Ожидаемый результат:** Название в задаче изменится.

**Фактический результат:** Название задачи остаётся прежним.

**Приоритет:** medium.

**Дополнительные материалы:** Имя файла «Название1» и «Название2»

## Баг 3

**Заголовок:** Нельзя создать задачу без заполнения поля «Описание».

**Описание:** Нельзя создать задачу без заполнения поля «Описание», хотя поле отмечено как необязательное (кнопка «Создать» работает, но задача не сохраняется).

**Предусловие:** Создание новой задачи.

**Окружение:** Windows 10, Chrome 143, разрешение 1920x1080.

**Шаги воспроизведения:**
1. В таск-трекере выбрать «Создать задачу».
2. Заполнить обязательные поля в задаче (название, проект, приоритет, статус (автоматически), исполнитель).
3. Нажать кнопку «Создать».
4. Дождаться результата.

**Ожидаемый результат:** Создастся новая задача со всей введённой информацией.

**Фактический результат:** Новая задача не создаётся.

**Приоритет:** high.

**Дополнительные материалы:** Имя файла «Создание1» и «Создание2»

## Баг 4

**Заголовок:** При изменении статуса задачи на «Backlog» – её статус не меняется

**Описание:** При изменении статуса в существующей задаче на статус «Backlog», её статус остаётся неизменным.

**Предусловие:** Создана задача с любым статусом, кроме статуса «Backlog».

**Окружение:** Windows 10, Chrome 143, разрешение 1920x1080.

**Шаги воспроизведения:**
1. В списке задач нажать на любую задачу с любым статусом, кроме статуса «Backlog», например «Реализация тестов API».
2. Выбрать новый статус «Backlog».
3. Нажать кнопку «Обновить».
4. Дождаться результата.

**Ожидаемый результат:** Статус у задачи изменится на статус «Backlog».

**Фактический результат:** Статус остается без изменений.

**Приоритет:** high.

**Дополнительные материалы:** Имя файла «Корректировка статуса1» и «Корректировка статуса2»
